# Prolifer8 — Project Scope & Structure

> **Prolifer8** is a web-based PWA for short-form content promotion. Inspired by Instagram Reels, its core purpose is **content exposure maximisation** — getting every piece of uploaded content in front of as many relevant viewers as possible, powered by a tag-weighted recommendation algorithm, user engagement signals, and a credit/tier-based boost system.

---

## 1. Core Concept

| Term | Definition |
|------|-----------|
| **Post** | A piece of content uploaded by a user: photo, audio clip, or short video. |
| **Exposure Window** | The period during which the algorithm actively circulates a post. Ends when a view cap or time cap is reached. |
| **Credits** | In-app currency used to extend or boost a post's exposure window beyond the default tier allowance. |
| **Account Tier** | Free or paid subscription level that determines the default exposure window and upload limits. |
| **Tag Weight** | A normalised score assigned to each post tag that drives recommendation probability. |
| **User Tag Profile** | A decaying weighted tally of tags from a user's viewing history, used to personalise the feed. |

## 2. Content Upload Rules

| Content Type | Size Limit | Duration Limit |
|---|---|---|
| **Photo** | 2 MB | — |
| **Audio** | 8 MB | 5 minutes |
| **Video** | 50 MB | 3 minutes |

- A post can carry **up to 10 tags**, entered by the uploader at time of posting.
- A post has a title, optional caption/description, and media file.

---

## 3. Exposure & Boost System

### 3.1 Default Exposure Window (by Tier)

Each account tier grants a default maximum view count and/or active duration before a post stops being actively circulated. Exact limits are configured per tier in the admin panel.

| Tier | Description |
|------|-------------|
| **Free** | Baseline view cap and time window. |
| **Paid (Standard)** | Higher view cap and longer active window. |
| **Paid (Pro)** | Maximum default caps; priority feed placement. |

### 3.2 Credit Boost

- Users can allocate credits to a specific post to **extend** its exposure window (more views and/or more time).
- The more credits allocated, the longer and wider the post circulates.
- Credits can be purchased via Stripe or cryptocurrency.
- **Currency rate:** 1,000 credits = $1.00 USD.

### 3.3 Exposure Termination

A post stops active circulation when **either** condition is met first:
1. The post's view cap is reached.
2. The post's active time window expires.

After termination, the post remains on the creator's profile but is no longer pushed by the algorithm.

---

## 4. Tag Weighting Formula

When a user attaches tags to a post, the tags are ordered (position 1 = most important). Weights are computed as:

$$\text{offset} = 1 - \frac{1}{1 + N}$$

$$\text{baseRate} = \frac{1 - \text{offset}}{N}$$

$$\text{bonus}_i = \frac{\text{pos}_i}{\sum_{j=1}^{N} \text{pos}_j} \times \text{offset}$$

$$w_i = \text{baseRate} + \text{bonus}_i$$

Where $N$ = total number of tags, $\text{pos}_i$ = position index of tag $i$ (1-based, position 1 = highest weight).

All tag weights sum to 1. Tags at the **start** of the list receive higher weight than those at the end (linear gradient via the bonus term).

---

## 5. Recommendation Algorithm

### 5.1 User Tag Profile

- Each user maintains a **weighted tag tally** derived from their viewing history.
- Tags are added to the tally with their post-level weights each time the user views a post.
- The tally is synced:
  - **localStorage** — updated frequently (real-time / session).
  - **Database** — synced daily as a background job.
- **Daily decay:** Every day, each tag's tally weight is multiplied by `0.99` (i.e., 1% decay).
- Tags whose tally weight falls below `0.01` are pruned from the profile.

### 5.2 Feed Scoring

A candidate post's recommendation probability for a given user is influenced by:

| Signal | Effect |
|--------|--------|
| **Tag alignment** | Posts whose tags overlap strongly with the user's tag profile receive a higher selection probability. |
| **Quality / effort rating** | Higher community ratings increase feed probability. |
| **Flags** | Flagged posts receive reduced circulation; heavily flagged posts are suppressed. |
| **Randomness** | A random factor ensures some diversity and gives new/unproven posts a chance. |
| **Credit boost** | Boosted posts receive a persistent multiplier on their selection probability. |

The feed is generated by sampling from this weighted probability distribution across eligible posts within the user's exposure window.

## 6. User Engagement & Moderation

### 6.1 Ratings

Users can rate any post on two axes:

| Axis | Scale |
|------|-------|
| **Quality** | 1–5 stars (or 0–100%) |
| **Effort** | 1–5 stars (or 0–100%) |

Aggregate scores feed back into the recommendation algorithm.

### 6.2 Reactions

- **Like** — positive signal; increases algorithm weight.
- **Downvote** — negative signal; decreases algorithm weight.

### 6.3 Content Flags

Users can flag posts with one or more of the following reasons:

| Flag | Description |
|------|-------------|
| Fake / AI | Deceptive or AI-generated without disclosure |
| Plagiarised | Content taken from another creator without credit |
| Abuse | Harassment or targeted bullying |
| Mature | Adult content not properly tagged |
| Hazardous | Dangerous activities or instructions |
| Scam | Fraudulent promotions or misleading content |
| Spam | Repetitive or unsolicited commercial content |
| Offensive | Hate speech or discriminatory content |

- Posts accumulating flags beyond a threshold are **automatically suppressed** pending review.
- Repeated violations result in account strikes or suspension.
